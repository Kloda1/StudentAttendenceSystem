# إعداد نظام الحضور للتعامل مع 5000 مستخدم متزامن

## المتطلبات الأساسية

### 1. إعداد قاعدة البيانات (MySQL)

تأكد من استخدام MySQL بدلاً من SQLite في ملف `.env`:

```env
DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=your_database_name
DB_USERNAME=your_username
DB_PASSWORD=your_password
```

### 2. تشغيلmigration الفهارس

```bash
php artisan migrate
```

### 3. إعداد Redis للطوابير (موصى به للـ 5000 مستخدم)

```env
QUEUE_CONNECTION=redis
REDIS_CLIENT=phpredis
```

#### إذا لم يكن Redis متاحاً، يمكن استخدام قاعدة البيانات:

```env
QUEUE_CONNECTION=database
```

### 4. تشغيل معالج الطوابير

في بيئة الإنتاج، يجب تشغيل معالج الطوابير:

```bash
# إذا كان استخدام Redis
php artisan queue:work redis --queue=attendance --sleep=3 --tries=3 --timeout=30

# أو إذا كان استخدام قاعدة البيانات
php artisan queue:work database --queue=attendance --sleep=3 --tries=3 --timeout=30
```

### 5. لتشغيل التطوير مع الطوابير

```bash
# الطريقة الأولى: تشغيل يدوي
php artisan queue:work

# الطريقة الثانية: استخدام الأمر المختصر
composer run dev
```

## كيفية العمل

### عند إرسال طلب الحضور:

1. **الخطوة الأولى**: يتم قبول الطلب فوراً وإرسال رسالة "جاري المعالجة..."
2. **الخطوة الثانية**: يتم إضافة الطلب إلى طابور المعالجة
3. **الخطوة الثالثة**: يتم معالجة الطلب في الخلفية
4. **الخطوة الرابعة**: يتم عرض النتيجة (نجاح/فشل)

### معالجة الأخطاء:

- إذا فشل OTP: يتم عرض رسالة "رمز التحقق غير صحيح"
- إذا انتهت صلاحية الجلسة: يتم عرض رسالة "انتهت صلاحية الجلسة"
- إذا لم يكن الطالب مسجلاً: يتم عرض رسالة "الطالب غير موجود"
- إذا حضر الطالب بالفعل: يتم عرض رسالة "لقد قمت بتسجيل حضورك بالفعل"
